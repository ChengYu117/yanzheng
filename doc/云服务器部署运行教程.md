# 云服务器部署运行教程

更新时间：2026-04-30
当前主流程：全量 MISC 数据集 SAE 评估 -> 可解释性分析 -> 因果候选导出 -> RE/NonRE 因果验证

这份教程对应当前仓库里的云端脚本 `deploy/gce/`。目录名虽然叫 `gce`，但只要是 Ubuntu + NVIDIA GPU 的云服务器都可以使用。

## 1. 当前云端脚本做什么

推荐入口是一键全流程：

```text
run_full_pipeline.sh
  -> run_full_eval.sh
  -> run_interpretability.sh
  -> run_causal.sh
  -> run_ai_judge.sh    # 默认关闭
```

每个阶段也可以单独断点重跑。当前默认数据集是：

```bash
DATA_DIR=data/mi_quality_counseling_misc
CAUSAL_DATA_DIR=${DATA_DIR}
```

也就是说，SAE 和因果验证都默认使用全量 MISC `unit_text`。因果验证当前仍是 `RE vs NonRE` 主任务，但数据来源不再是旧的 `data/mi_re` 平衡集，而是从全量 MISC 中构造 `label_re`。

## 2. 推荐服务器配置

最低建议：

- Ubuntu 22.04 或 24.04
- 1 张 NVIDIA GPU
- 24GB 以上显存更稳
- 100GB 以上磁盘空间
- `nvidia-smi` 可用

先检查：

```bash
nvidia-smi
python3 --version
git --version
```

如果 `nvidia-smi` 不可用，先处理 GPU 驱动和 CUDA 环境，不要继续跑实验。

## 3. 获取项目代码

如果服务器还没有项目：

```bash
cd ~
git clone -b sae-re-project https://github.com/ChengYu117/yanzheng.git nlp-re
cd ~/nlp-re
```

如果服务器已经有项目：

```bash
cd ~/nlp-re
git fetch origin
git checkout sae-re-project
git pull --ff-only origin sae-re-project
```

如果服务器没有 GitHub 权限，可以在本地打包后上传：

```powershell
cd D:\project\NLP_re_dataset_model_base
python package_project.py
```

上传生成的压缩包到服务器后解压即可。不要上传 `outputs/`、模型目录、缓存目录或日志目录。

## 4. 配置 `.env`

第一次运行时复制模板：

```bash
cd ~/nlp-re
cp -n deploy/gce/env.example deploy/gce/.env
nano deploy/gce/.env
```

正式全量 MISC 流程建议至少确认这些字段：

```bash
DATA_ROOT=/mnt/disks/data
MODEL_DIR=${DATA_ROOT}/models/Llama-3.1-8B
HF_HOME=${DATA_ROOT}/hf-cache
OUTPUT_ROOT=${DATA_ROOT}/outputs

PYTHON_BIN=python3.11
PYTORCH_INDEX_URL=https://download.pytorch.org/whl/cu124

# 如果使用已有 conda 环境，填写环境名；否则留空并使用 .venv。
CONDA_ENV_NAME=qwen-env-py311
# 如果脚本找不到 conda，再显式填写，例如：
# CONDA_BASE=/root/miniconda3

DATA_DIR=data/mi_quality_counseling_misc
CAUSAL_DATA_DIR=${DATA_DIR}
ALLOW_LEGACY_CAUSAL_DATA=0
DEVICE=cuda
CHECKPOINT_TOPK_SEMANTICS=hard

HF_TOKEN=<你的 Hugging Face token>
MODEL_HF_REPO_ID=meta-llama/Llama-3.1-8B
MODEL_REVISION=main
HF_ENDPOINT=https://hf-mirror.com
HF_HUB_DOWNLOAD_TIMEOUT=60
HF_HUB_ETAG_TIMEOUT=60
HF_HUB_DOWNLOAD_MAX_WORKERS=1

SAE_OUTPUT_DIR=${OUTPUT_ROOT}/sae_eval_full
CAUSAL_OUTPUT_DIR=${OUTPUT_ROOT}/causal_validation_full
PIPELINE_OUTPUT_DIR=${OUTPUT_ROOT}/misc_full_pipeline
MAPPING_OUTPUT_DIR=${SAE_OUTPUT_DIR}/interpretability/mapping_structure
FOLLOWUP_OUTPUT_DIR=${SAE_OUTPUT_DIR}/interpretability/followup_analysis
CAUSAL_CANDIDATE_OUTPUT_DIR=${SAE_OUTPUT_DIR}/interpretability/causal_candidates
CAUSAL_LABEL=RE

RUN_SAE=1
RUN_INTERPRETABILITY=1
RUN_CAUSAL=1
RUN_AI_JUDGE=0

SAE_BATCH_SIZE=4
SAE_MAX_SEQ_LEN=128
CAUSAL_BATCH_SIZE=4
CAUSAL_MAX_SEQ_LEN=128
CAUSAL_LAMBDAS="0.5 1.0 1.5 2.0"
```

说明：

- `CONDA_ENV_NAME` 不为空时，脚本会自动激活该 conda 环境。
- `CONDA_ENV_NAME` 为空时，脚本会使用 `${VENV_DIR}` 创建并激活虚拟环境。
- 正式流程请保持 `DATA_DIR=data/mi_quality_counseling_misc`。
- 正式因果验证请保持 `CAUSAL_DATA_DIR=${DATA_DIR}` 和 `ALLOW_LEGACY_CAUSAL_DATA=0`。
- 不要把真实 `HF_TOKEN` 提交到 Git。

## 5. 初始化运行环境

执行：

```bash
cd ~/nlp-re
bash deploy/gce/bootstrap.sh
```

这个脚本会：

- 安装 `tmux`、`git`
- 使用 conda 环境或创建 `.venv`
- 安装 PyTorch
- 安装 `requirements.txt`
- 执行 `pip install -e .`
- 验证 `lm-saes`、`transformer-lens`、`sentencepiece`、`tiktoken`
- 验证主要 Python 入口的 `--help`

成功时会看到：

```text
Bootstrap completed successfully.
```

如果报错：

```text
ensurepip is not available
```

执行：

```bash
sudo apt-get update
sudo apt-get install -y python3.11-venv
rm -rf .venv
bash deploy/gce/bootstrap.sh
```

如果找不到 `python3.11-venv`，再加 deadsnakes 源：

```bash
sudo apt-get update
sudo apt-get install -y software-properties-common
sudo add-apt-repository -y ppa:deadsnakes/ppa
sudo apt-get update
sudo apt-get install -y python3.11 python3.11-venv
rm -rf .venv
bash deploy/gce/bootstrap.sh
```

## 6. 下载基础模型

先确认 `.env` 中有：

```bash
HF_TOKEN=<你的 Hugging Face token>
HF_ENDPOINT=https://hf-mirror.com
HF_HUB_DOWNLOAD_MAX_WORKERS=1
```

下载：

```bash
cd ~/nlp-re
bash deploy/gce/download_model.sh
```

默认下载到：

```bash
/mnt/disks/data/models/Llama-3.1-8B
```

检查模型目录：

```bash
source deploy/gce/common.sh
load_gce_env
ls -lh "$MODEL_DIR" | head
ls "$MODEL_DIR"/tokenizer* "$MODEL_DIR"/config.json 2>/dev/null
```

Llama 3.1 本地目录至少应该包含类似文件：

```text
config.json
tokenizer.json
tokenizer_config.json
special_tokens_map.json
model-*.safetensors
```

如果下载卡住，降低并发后重试：

```bash
cd ~/nlp-re
export HF_ENDPOINT=https://hf-mirror.com
export HF_HUB_DOWNLOAD_MAX_WORKERS=1
bash deploy/gce/download_model.sh
```

如果报错栈里还出现 `login(token=...)`，说明服务器上的 `download_model.sh` 不是当前新版脚本。先更新代码：

```bash
cd ~/nlp-re
git fetch origin
git pull --ff-only origin sae-re-project
grep -n "login(" deploy/gce/download_model.sh || true
grep -n "snapshot_download" deploy/gce/download_model.sh
```

当前新版脚本不调用 `login()`，而是直接把 token 传给 `snapshot_download()`。

## 9. 分阶段手动运行

如果不想使用一键总流程，也可以按阶段手动运行。

SAE 全量评估：

```bash
bash deploy/gce/run_full_eval.sh
```

等价核心命令：

```bash
python -u run_sae_evaluation.py \
  --model-dir "${MODEL_DIR}" \
  --device "${DEVICE}" \
  --data-dir "${DATA_DIR}" \
  --data-format auto \
  --label-mode misc_multilabel \
  --output-dir "${SAE_OUTPUT_DIR}" \
  --batch-size "${SAE_BATCH_SIZE}" \
  --max-seq-len "${SAE_MAX_SEQ_LEN}" \
  --checkpoint-topk-semantics "${CHECKPOINT_TOPK_SEMANTICS}" \
  --full-structural
```

可解释性阶段：

```bash
bash deploy/gce/run_interpretability.sh
```

该阶段会依次运行：

```text
run_misc_mapping_structure_analysis.py
run_misc_interpretability_analysis.py
run_misc_causal_candidate_export.py
```

因果验证：

```bash
bash deploy/gce/run_causal.sh
```

等价核心命令：

```bash
python -u causal/run_experiment.py \
  --model-dir "${MODEL_DIR}" \
  --device "${DEVICE}" \
  --candidate-csv "${CAUSAL_CANDIDATE_OUTPUT_DIR}/label_candidates/${CAUSAL_LABEL}_candidate_latents.csv" \
  --data-dir "${CAUSAL_DATA_DIR}" \
  --output-dir "${CAUSAL_OUTPUT_DIR}" \
  --batch-size "${CAUSAL_BATCH_SIZE}" \
  --max-seq-len "${CAUSAL_MAX_SEQ_LEN}" \
  --checkpoint-topk-semantics "${CHECKPOINT_TOPK_SEMANTICS}" \
  --lambdas 0.5 1.0 1.5 2.0
```

注意：不要给 `deploy/gce/run_causal.sh` 额外追加 `--data-dir`。要换因果数据目录，请改 `.env` 的 `CAUSAL_DATA_DIR`，或用环境变量覆盖。

## 11. 因果验证的数据口径

当前云端因果验证默认跑：

```bash
CAUSAL_DATA_DIR=data/mi_quality_counseling_misc
CAUSAL_LABEL=RE
```

含义：

- 使用全量 MISC 行为单元。
- 由统一数据读取器构造 `label_re`。
- 因果验证任务仍是 `RE vs NonRE`。
- 候选 latent 优先来自解释性阶段导出的 `${CAUSAL_LABEL}_candidate_latents.csv`。

`run_causal.sh` 默认会拒绝这些旧路径：

```text
data/mi_re
data/cactus
derived/re_nonre
```

如果你明确要做旧实验或对照实验，需要显式允许：

```bash
ALLOW_LEGACY_CAUSAL_DATA=1 \
CAUSAL_DATA_DIR=data/mi_re \
bash deploy/gce/run_causal.sh
```

正式报告口径以全量 `data/mi_quality_counseling_misc` 为准。

## 13. 常见问题

### 13.1 `Candidate CSV not found`

说明因果阶段找不到候选 latent。先确认解释性阶段已经完成：

```bash
ls "${CAUSAL_CANDIDATE_OUTPUT_DIR}/label_candidates/${CAUSAL_LABEL}_candidate_latents.csv"
```

如果不存在，先跑：

```bash
RUN_SAE=0 RUN_CAUSAL=0 bash deploy/gce/run_full_pipeline.sh
```

### 13.2 `Couldn't instantiate the backend tokenizer`

先检查依赖：

```bash
source deploy/gce/common.sh
load_gce_env
activate_venv
python - <<'PY'
import importlib.util
for name in ("sentencepiece", "tiktoken", "google.protobuf"):
    print(name, importlib.util.find_spec(name) is not None)
PY
```

缺少时安装：

```bash
python -m pip install -U sentencepiece tiktoken protobuf
```

再检查模型目录是否完整：

```bash
ls "$MODEL_DIR"/tokenizer* "$MODEL_DIR"/config.json 2>/dev/null
```

如果缺少 tokenizer 或权重文件，重新下载模型。

### 13.3 CUDA 显存不足

在 `.env` 中调小：

```bash
SAE_BATCH_SIZE=2
CAUSAL_BATCH_SIZE=2
SAE_MAX_SEQ_LEN=128
CAUSAL_MAX_SEQ_LEN=128
```

如果还不够，继续调到 `1`。

### 13.4 tmux session 已存在

查看：

```bash
tmux ls
```

进入已有 session：

```bash
tmux attach -t misc_full_pipeline
```

如果确认旧任务已经结束，可以关闭：

```bash
tmux kill-session -t misc_full_pipeline
```

### 13.5 任务中断但没有明显错误

按顺序看：

```bash
cat "${PIPELINE_OUTPUT_DIR}/pipeline_status.json"
tail -n 100 "${PIPELINE_OUTPUT_DIR}/pipeline.log"
tail -n 100 "${CAUSAL_OUTPUT_DIR}/run.log"
cat "${CAUSAL_OUTPUT_DIR}/run_status.json" 2>/dev/null || true
dmesg -T | tail -n 80
```

如果 `dmesg` 里有 OOM kill，优先降低 batch size。

### 13.6 服务器 `git pull` 有本地改动冲突

先查看：

```bash
cd ~/nlp-re
git status --short
```

优先把 `.env` 备份出来，然后把服务器本地改动保存到 stash：

```bash
cp deploy/gce/.env ~/gce.env.backup.$(date +%Y%m%d_%H%M%S)
git fetch origin
git stash push -u -m "server local changes before cloud script update"
git pull --ff-only origin sae-re-project
cp ~/gce.env.backup.* deploy/gce/.env
```

如果 `git pull --ff-only` 仍然失败，建议重新 clone 到新目录，再把 `.env` 复制过去：

```bash
cd ~
git clone -b sae-re-project https://github.com/ChengYu117/yanzheng.git nlp-re-new
cp ~/gce.env.backup.* ~/nlp-re-new/deploy/gce/.env
```
