# 云服务器部署运行教程

本教程面向“把当前项目部署到一台 Linux GPU 云服务器并跑完整流程”的场景。  
仓库里已经自带了 `deploy/gce/` 脚本，这份文档把这些脚本串成一条可执行流程。

适用范围：

- 单机 Linux GPU 服务器
- 已安装 NVIDIA 驱动，`nvidia-smi` 可用
- Python 3.10
- 适合 Google Compute Engine，也适合大多数 Ubuntu GPU 服务器

---

## 1. 项目里哪些脚本是云部署入口

核心入口：

- `run_sae_evaluation.py`
- `causal/run_experiment.py`
- `run_ai_re_judge.py`

云部署包装脚本：

- `deploy/gce/common.sh`
- `deploy/gce/bootstrap.sh`
- `deploy/gce/download_model.sh`
- `deploy/gce/run_full_eval.sh`
- `deploy/gce/run_causal.sh`
- `deploy/gce/start_full_eval_tmux.sh`
- `deploy/gce/start_causal_tmux.sh`

这意味着：

- 项目本体不是“只能在云上跑”
- 只是额外提供了一套适合云服务器的启动和目录约定

---

## 2. 推荐服务器配置

最低建议：

- Ubuntu 22.04
- 1 张 NVIDIA GPU
- 24GB 显存更稳
- 100GB 以上磁盘

更稳的配置：

- 1 x L4 / 1 x A10 / 1 x A100
- 200GB 磁盘

先检查服务器是否正常：

```bash
nvidia-smi
python3 --version
git --version
```

如果 `nvidia-smi` 不通，先不要继续部署。

---

## 3. 上传项目到云服务器

你可以用两种方式：

### 方式 A：直接 git clone

如果仓库已经在 GitHub 上：

```bash
git clone <你的仓库地址> ~/nlp-re
cd ~/nlp-re
```

### 方式 B：上传源码包

如果你本地还没推到 GitHub，可以先在本地打包：

```powershell
python package_project.py
```

然后把生成的 `tar.gz` 上传到云服务器，解压到：

```bash
mkdir -p ~/nlp-re
tar -xzf 你的源码包.tar.gz -C ~/nlp-re
cd ~/nlp-re
```

---

## 4. 配置云端环境变量

先复制模板：

```bash
cp deploy/gce/env.example deploy/gce/.env
```

编辑：

```bash
nano deploy/gce/.env
```

至少确认这些项：

```bash
DATA_ROOT=/mnt/disks/data
MODEL_DIR=${DATA_ROOT}/models/Llama-3.1-8B
HF_HOME=${DATA_ROOT}/hf-cache
OUTPUT_ROOT=${DATA_ROOT}/outputs

VENV_DIR=${HOME}/nlp-re/.venv
PYTHON_BIN=python3.10
PYTORCH_INDEX_URL=https://download.pytorch.org/whl/cu124

HF_TOKEN=你的 Hugging Face token
MODEL_HF_REPO_ID=meta-llama/Llama-3.1-8B
MODEL_REVISION=main

SAE_OUTPUT_DIR=${OUTPUT_ROOT}/sae_eval_full
CAUSAL_OUTPUT_DIR=${OUTPUT_ROOT}/causal_validation_full
```

如果你的服务器没有挂载 `/mnt/disks/data`，改成家目录也可以，例如：

```bash
DATA_ROOT=${HOME}/data
```

---

## 5. 初始化环境

执行：

```bash
bash deploy/gce/bootstrap.sh
```

这个脚本会做这些事：

- 安装 Python、pip、tmux、git
- 创建虚拟环境
- 安装 PyTorch
- 安装项目依赖
- `pip install -e .`
- 验证三个主入口能启动

成功后你应该能看到：

- `Bootstrap completed successfully.`

如果失败，优先检查：

- `deploy/gce/.env` 是否存在
- `python3.10` 是否能安装
- GPU 驱动是否正常

---

## 6. 下载基础模型

执行：

```bash
bash deploy/gce/download_model.sh
```

模型会下载到：

- `${MODEL_DIR}`

默认是：

- `/mnt/disks/data/models/Llama-3.1-8B`

这一步需要：

- 有效的 `HF_TOKEN`
- 你的账号对 `meta-llama/Llama-3.1-8B` 有访问权限

---

## 7. 先做最小检查

先验证环境加载正常：

```bash
source deploy/gce/common.sh
load_gce_env
echo "$MODEL_DIR"
echo "$OUTPUT_ROOT"
echo "$SAE_OUTPUT_DIR"
echo "$CAUSAL_OUTPUT_DIR"
```

再检查入口：

```bash
source "${VENV_DIR}/bin/activate"
python run_sae_evaluation.py --help
python causal/run_experiment.py --help
python run_ai_re_judge.py --help
```

---

## 8. 跑 SAE 主流程

### 直接前台跑

```bash
bash deploy/gce/run_full_eval.sh
```

### 推荐：用 tmux 后台跑

```bash
bash deploy/gce/start_full_eval_tmux.sh
```

或者手动：

```bash
tmux new -d -s sae_eval 'cd ~/nlp-re && bash deploy/gce/run_full_eval.sh'
```

查看状态：

```bash
tmux ls
tmux attach -t sae_eval
```

主流程日志：

```bash
tail -f ${SAE_OUTPUT_DIR}/run.log
```

### 当前云端脚本默认跑哪份数据

注意这一点：

- `deploy/gce/run_full_eval.sh` 当前写死的是：
  - `--data-dir "data/mi_re"`

也就是说，默认云端包装脚本跑的是旧的 `mi_re` 数据，而不是 `cactus`。

如果你要跑 `cactus`，直接这样：

```bash
bash deploy/gce/run_full_eval.sh --data-dir data/cactus
```

---

## 9. 跑因果验证流程

先确保 SAE 主流程已经跑完，并且有：

- `${SAE_OUTPUT_DIR}/candidate_latents.csv`

然后执行：

```bash
bash deploy/gce/run_causal.sh
```

推荐后台跑：

```bash
bash deploy/gce/start_causal_tmux.sh
```

日志：

```bash
tail -f ${CAUSAL_OUTPUT_DIR}/run.log
```

当前云端因果脚本同样默认用的是：

- `--data-dir "data/mi_re"`

如果你要让因果流程也跑 `cactus`：

```bash
bash deploy/gce/run_causal.sh --data-dir data/cactus
```

---

## 10. 跑 AI 专家代理评审

如果你要在云端跑 judge，还要在 `deploy/gce/.env` 里补：

```bash
OPENAI_API_KEY=...
OPENAI_BASE_URL=...
OPENAI_MODEL=...
```

然后激活环境并执行：

```bash
source deploy/gce/common.sh
load_gce_env
source "${VENV_DIR}/bin/activate"
cd ~/nlp-re

python run_ai_re_judge.py \
  --input-dir "${SAE_OUTPUT_DIR}" \
  --output-dir "${SAE_OUTPUT_DIR}/ai_judge" \
  --model "${OPENAI_MODEL}"
```

如果是百炼兼容接口，还需要按你本地那套配置传兼容参数。

## 11. 结果目录

默认输出：

- SAE 主流程：
  - `${SAE_OUTPUT_DIR}`
- 因果流程：
  - `${CAUSAL_OUTPUT_DIR}`

如果使用默认 `env.example`，通常就是：

- `/mnt/disks/data/outputs/sae_eval_full`
- `/mnt/disks/data/outputs/causal_validation_full`

常见结果文件：

### SAE 主流程

- `metrics_structural.json`
- `metrics_ce_kl.json`
- `metrics_functional.json`
- `candidate_latents.csv`
- `judge_bundle/`
- `latent_cards/`
- `run.log`

### 因果流程

- `selected_groups.json`
- `results_necessity.json`
- `results_sufficiency.json`
- `results_selectivity.json`
- `results_group.json`
- `summary_tables.md`
- `run.log`

## 12. SSH 断线怎么办

不要直接在裸 SSH 会话里跑长任务。推荐：

- `tmux`

常用命令：

```bash
tmux ls
tmux attach -t sae_eval
tmux attach -t causal_eval
```

如果你已经用包装脚本跑了：

```bash
bash deploy/gce/start_full_eval_tmux.sh
bash deploy/gce/start_causal_tmux.sh
```

SSH 断开后任务仍会继续。

## 13. 常见问题

### 1. `HF_TOKEN is empty`

说明你还没在：

- `deploy/gce/.env`

里填 token。

### 2. `Candidate CSV not found`

说明因果流程启动时没找到：

- `${SAE_OUTPUT_DIR}/candidate_latents.csv`

先跑：

```bash
bash deploy/gce/run_full_eval.sh
```

### 3. 想换输出目录

可以直接改：

- `deploy/gce/.env`

里的：

- `SAE_OUTPUT_DIR`
- `CAUSAL_OUTPUT_DIR`

也可以命令行覆盖主入口脚本的 `--output-dir`。

### 4. 想换数据集

当前云端包装脚本默认是 `data/mi_re`。  
如果你已经转到 `cactus`，运行时要显式加：

```bash
--data-dir data/cactus
```

---

## 14. 建议的完整执行顺序

推荐你在新服务器上按这个顺序跑：

```bash
cd ~/nlp-re
cp deploy/gce/env.example deploy/gce/.env
nano deploy/gce/.env

bash deploy/gce/bootstrap.sh
bash deploy/gce/download_model.sh

bash deploy/gce/start_full_eval_tmux.sh --data-dir data/cactus
bash deploy/gce/start_causal_tmux.sh --data-dir data/cactus
```

如果你暂时还沿用旧数据，就把 `data/cactus` 去掉，保持脚本默认行为。

---

## 15. 一句话总结

部署这个项目到云服务器，本质上就是 5 步：

1. 上传代码
2. 填好 `deploy/gce/.env`
3. 跑 `bootstrap.sh`
4. 跑 `download_model.sh`
5. 用 `run_full_eval.sh` 和 `run_causal.sh` 跑实验

其中最容易出错的地方只有两个：

- 模型权限（`HF_TOKEN`）
- 数据集路径（当前 GCE 包装脚本默认还是 `data/mi_re`）
