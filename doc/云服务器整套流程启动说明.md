# 云服务器整套流程启动说明

这份文档对应当前仓库里已经落地的主流程，目标是让你从一台新云服务器出发，按固定顺序跑通：

1. 环境初始化
2. 模型下载
3. SAE 主评估
4. 因果验证
5. 可选的 AI judge 复核

当前推荐口径：

- `SAE` 推理模式：`aligned_datasetwise`
- 结构评估：开启 `--full-structural`
- 因果评估组：`G1 / G5 / G10 / G20`

## 1. 前提条件

建议云服务器满足以下条件：

- Linux，推荐 `Ubuntu 22.04`
- 已正确安装 `NVIDIA Driver` 和 `CUDA`
- `nvidia-smi` 可正常执行
- 代码仓库已经上传到服务器
- 你已经准备好 Hugging Face 访问令牌

如果仓库还没上传，可以先在本地打包再上传，或者直接在服务器上 `git clone`。

## 2. 推荐的执行方式

仓库现在有两套入口：

- 云服务器封装脚本：`deploy/gce/*.sh`
- 直接 Python 脚本：
  - `run_sae_evaluation.py`
  - `causal/run_experiment.py`
  - `run_ai_re_judge.py`

推荐优先使用 `deploy/gce` 封装脚本。原因是：

- 会统一读取 `deploy/gce/.env`
- 会自动带上推荐参数
- 更不容易把 `legacy` 和 `aligned_datasetwise` 跑混
- 输出目录更稳定

## 3. 一次性生成环境配置

在仓库根目录执行：

```bash
bash deploy/gce/configure_env.sh \
  --hf-token YOUR_HF_TOKEN \
  --data-root /mnt/disks/data \
  --sae-inference-mode aligned_datasetwise
```

这一步会生成：

```text
deploy/gce/.env
```

默认会写入这些关键路径：

- `MODEL_DIR=/mnt/disks/data/models/Llama-3.1-8B`
- `HF_HOME=/mnt/disks/data/hf-cache`
- `OUTPUT_ROOT=/mnt/disks/data/outputs`
- `SAE_OUTPUT_DIR=/mnt/disks/data/outputs/sae_eval_full`
- `CAUSAL_OUTPUT_DIR=/mnt/disks/data/outputs/causal_validation_full`

如果你想一开始就把参数定好，也可以直接在这里一起传：

```bash
bash deploy/gce/configure_env.sh \
  --hf-token YOUR_HF_TOKEN \
  --data-root /mnt/disks/data \
  --model-dir /mnt/disks/data/models/Llama-3.1-8B \
  --sae-batch-size 4 \
  --causal-batch-size 4 \
  --causal-n-bootstrap 10 \
  --causal-side-effect-max-samples 16 \
  --sae-compare-mean 0 \
  --sae-inference-mode aligned_datasetwise
```

## 4. 初始化 Python 环境

执行：

```bash
bash deploy/gce/bootstrap.sh
```

这个脚本会自动完成：

- 安装系统依赖
- 创建虚拟环境
- 安装 `torch`
- 安装项目依赖
- 校验三个主入口脚本能否启动：
  - `run_sae_evaluation.py`
  - `causal/run_experiment.py`
  - `run_ai_re_judge.py`

如果这一步成功，说明运行环境已经准备好。

## 5. 下载基础模型

执行：

```bash
bash deploy/gce/download_model.sh
```

默认会把底模下载到：

```text
/mnt/disks/data/models/Llama-3.1-8B
```

如果你在 `configure_env.sh` 里改了 `--model-dir`，这里会落到你指定的位置。

## 6. 启动 SAE 主评估

建议放在 `tmux` 里跑：

```bash
tmux new -s sae
bash deploy/gce/run_full_eval.sh
```

当前这个封装脚本默认会带上：

- `--full-structural`
- `--sae-inference-mode "${SAE_INFERENCE_MODE}"`

如果 `.env` 是按推荐方式生成的，那么这里实际跑的是：

- `aligned_datasetwise`
- 全量结构统计

默认输出目录：

```text
/mnt/disks/data/outputs/sae_eval_full
```

关键输出文件包括：

- `metrics_structural.json`
- `metrics_ce_kl.json`
- `metrics_functional.json`
- `candidate_latents.csv`
- `judge_bundle/`
- `run.log`

如果你想比较 `max / mean` 两种 pooling，有两种方式：

在生成 `.env` 时打开：

```bash
--sae-compare-mean 1
```

或者临时加参数：

```bash
bash deploy/gce/run_full_eval.sh --compare-mean
```

## 7. 启动因果验证

确认上一步已经产出：

```text
${SAE_OUTPUT_DIR}/candidate_latents.csv
```

然后执行：

```bash
tmux new -s causal
bash deploy/gce/run_causal.sh
```

默认输出目录：

```text
/mnt/disks/data/outputs/causal_validation_full
```

这个脚本会自动带上：

- `--sae-inference-mode "${SAE_INFERENCE_MODE}"`
- `--n-bootstrap "${CAUSAL_N_BOOTSTRAP}"`
- `--side-effect-max-samples "${CAUSAL_SIDE_EFFECT_MAX_SAMPLES}"`

当前因果流程会评估：

- `G1`
- `G5`
- `G10`
- `G20`

关键输出文件包括：

- `selected_groups.json`
- `results_necessity.json`
- `results_sufficiency.json`
- `results_selectivity.json`
- `results_group.json`
- `summary_tables.md`
- `run.log`

如果你想比较不同 pooling，可以加：

```bash
bash deploy/gce/run_causal.sh --compare-pooling
```

这样会额外生成：

- `pooling_comparison.json`
- `pooling_comparison.md`

## 8. 可选：运行 AI Judge

主评估完成后，`SAE` 输出目录里会自动生成：

```text
${SAE_OUTPUT_DIR}/judge_bundle
```

如果你要做 AI 复核，可以执行：

```bash
source deploy/gce/.env
source "${VENV_DIR}/bin/activate"
python run_ai_re_judge.py \
  --input-dir "${SAE_OUTPUT_DIR}" \
  --output-dir "${OUTPUT_ROOT}/ai_judge_full" \
  --groups G1,G5,G10,G20
```

如果你只是想先检查 prompt，不实际调用 API：

```bash
python run_ai_re_judge.py \
  --input-dir "${SAE_OUTPUT_DIR}" \
  --output-dir "${OUTPUT_ROOT}/ai_judge_dryrun" \
  --groups G1,G5,G10,G20 \
  --dry-run-prompts
```

AI judge 依赖这些环境变量之一已经可用：

- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`（如果你走兼容接口）
- `OPENAI_MODEL`

## 10. 临时覆盖参数的常见方式

如果你不想重写 `.env`，可以在启动时临时覆盖。

临时改主评估输出目录：

```bash
SAE_OUTPUT_DIR=/mnt/disks/data/outputs/sae_eval_full_v2 \
bash deploy/gce/run_full_eval.sh
```

临时改因果输出目录：

```bash
CAUSAL_OUTPUT_DIR=/mnt/disks/data/outputs/causal_validation_full_v2 \
bash deploy/gce/run_causal.sh
```

临时改 bootstrap 次数：

```bash
CAUSAL_N_BOOTSTRAP=20 \
bash deploy/gce/run_causal.sh
```

临时切回旧口径：

```bash
SAE_INFERENCE_MODE=legacy \
bash deploy/gce/run_full_eval.sh
```

或者：

```bash
SAE_INFERENCE_MODE=legacy \
bash deploy/gce/run_causal.sh
```

## 11. 不用封装脚本，直接跑 Python

如果你不想走 `deploy/gce`，也可以直接在仓库根目录跑。

先激活你的环境，然后主评估：

```bash
python run_sae_evaluation.py \
  --model-dir /mnt/disks/data/models/Llama-3.1-8B \
  --data-dir data/mi_re \
  --output-dir /mnt/disks/data/outputs/sae_eval_full \
  --batch-size 4 \
  --max-seq-len 128 \
  --sae-inference-mode aligned_datasetwise \
  --full-structural
```

因果验证：

```bash
python causal/run_experiment.py \
  --model-dir /mnt/disks/data/models/Llama-3.1-8B \
  --candidate-csv /mnt/disks/data/outputs/sae_eval_full/candidate_latents.csv \
  --data-dir data/mi_re \
  --output-dir /mnt/disks/data/outputs/causal_validation_full \
  --batch-size 4 \
  --max-seq-len 128 \
  --sae-inference-mode aligned_datasetwise \
  --n-bootstrap 10 \
  --side-effect-max-samples 16
```

可选 AI judge：

```bash
python run_ai_re_judge.py \
  --input-dir /mnt/disks/data/outputs/sae_eval_full \
  --output-dir /mnt/disks/data/outputs/ai_judge_full \
  --groups G1,G5,G10,G20
```

## 12. 验收清单

建议至少确认这些点：

- `nvidia-smi` 正常
- `deploy/gce/.env` 已生成
- `bash deploy/gce/bootstrap.sh` 成功
- `MODEL_DIR` 下已存在完整模型
- `SAE_OUTPUT_DIR/run.log` 已开始写入
- `CAUSAL_OUTPUT_DIR/run.log` 已开始写入
- `metrics_structural.json` 里出现：
  - `structural_scope`
  - `sae_inference_mode`
  - `space_metrics`
- `selected_groups.json` 里出现：
  - `G10`
  - `sae_inference_mode`
